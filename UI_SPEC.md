# UI SPEC (Do Not Change)

このリポジトリのUI/UX仕様は本書が唯一の正です。
**Claude Code / Codex など自動生成ツールは、UIの見た目・文言・初期値・ホットキー・レイアウトを勝手に変更してはいけません。**

## UI変更ポリシー
- UI変更（文言、配置、表示/非表示、初期値、ホットキー、画面遷移、設定項目名）を行う場合：
  1) まず `UI_SPEC.md` と `ui_contract.json` に変更案を「提案」として追記する（コードは変更しない）
  2) 人間のレビュー承認後にのみ実装する
- 承認なしのUI変更はバグとして扱い、差し戻します。

---

## 対象アプリ
- 名称（表示名）: Auto Clicker (Safe)
- 対応OS: Windows 10 / 11
- UI: 1画面（メインウィンドウ） + 任意でタスクトレイ常駐
- 設定保存: JSON（例 `settings.json`）で次回起動時に復元

---

## メインウィンドウ仕様

### ウィンドウ
- タイトル: `Auto Clicker (Safe)`
- 最小サイズ: 幅 520 / 高さ 520（目安）
- リサイズ: 可
- 最前面固定: なし（将来追加は提案扱い）
- 閉じる(X)動作:
  - `常駐: ON` の場合: トレイへ最小化（プロセス継続）
  - `常駐: OFF` の場合: 終了（実行中なら停止してから終了）

### 画面レイアウト（上から順）
#### 1) ステータスバー（上部）
- 表示項目:
  - 状態ラベル: `状態: 停止中` / `状態: 実行中`
  - モードラベル: `モード: 単一点` / `モード: 複数点`
  - 次クリックまで（任意）: `次まで: 123ms`（実装できる場合のみ。UI文言は固定）
- 状態表示の色変更などは不要（将来変更は提案扱い）

#### 2) モード選択
- ラジオボタン2つ
  - `単一点`
  - `複数点`
- 選択により、下部の入力エリアが切り替わる（表示/非表示）

#### 3) 共通設定（両モード共通）
- クリック種別:
  - ドロップダウン: `左クリック` / `右クリック` / `中クリック`
  - 初期値: `左クリック`
- 間隔(ms):
  - 数値入力（整数）
  - 範囲: 1〜600000
  - 初期値: 100
- 回数:
  - ラジオ: `無限` / `指定回数`
  - `指定回数`選択時の数値入力（整数）
  - 範囲: 1〜1000000
  - 初期値: 無限

#### 4) 単一点モード設定（単一点選択時のみ表示）
- 座標:
  - `X:` 数値（整数）
  - `Y:` 数値（整数）
  - 初期値: 0, 0
- ボタン:
  - `現在位置を取得`（押した瞬間のマウス位置をX/Yへ反映）
  - `3秒後に取得`（押下→3秒カウントダウン→取得してX/Yへ反映）
- 注記（固定文言）:
  - `※ マルチモニタ環境では座標が負になる場合があります。`

#### 5) 複数点モード設定（複数点選択時のみ表示）
- 座標リスト（テーブル/リスト）
  - 列: `#` / `X` / `Y` / `待機(ms)`
  - `待機(ms)` は各点クリック後の追加待機（未指定なら0）
  - 初期値: 空
- 追加・削除・並び替えボタン（順番固定）
  - `追加(現在位置)`
  - `追加(3秒後)`
  - `削除`
  - `上へ`
  - `下へ`
- `削除/上へ/下へ` は選択行がない場合は無効（押せない）
- 複数点実行ルール:
  - 1周 = リスト先頭から末尾まで1回ずつクリック
  - 「回数: 指定回数」の回数は **周回数** を意味する
  - 各点クリック後: `待機(ms)` + 共通 `間隔(ms)` を待って次へ

#### 6) ホットキー設定
- 表示項目（固定）
  - `開始/停止: F6`
  - `停止: F7`
- 変更UI:
  - 各項目に `変更` ボタン
  - クリック後「次に押したキーを割り当て」モードに入り、1キー入力で確定
  - ESCでキャンセル（割り当て変更しない）
- 禁止:
  - 同一ホットキーの重複割り当ては不可（エラー表示）
  - 予約キー（例: Winキー単体）は不可（エラー表示）

#### 7) 操作ボタン（最下部）
- `開始`（実行中は無効）
- `停止`（停止中は無効）
- `終了`（常駐ONでも「終了」は必ずプロセス終了）

#### 8) 常駐（任意）
- チェックボックス: `タスクトレイに常駐する`
- 初期値: ON
- トレイメニュー（固定文言）:
  - `表示`
  - `開始/停止`
  - `停止`
  - `終了`

## エラー/バリデーション表示
- 入力値不正時は開始できない
- 表示方法は以下のどちらか（どちらを採用してもUI文言は固定）
  - 各入力欄の直下にエラーテキスト
  - またはメッセージボックス
- エラー文言（固定）
  - `入力値を確認してください。`
  - `ホットキーが重複しています。別のキーにしてください。`
  - `ホットキーとして使用できないキーです。`
