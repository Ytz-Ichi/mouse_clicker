# Auto Clicker (Safe)

Windows用オートクリッカー（.NET 8 / WPF）

## 機能

- **単一点モード**: 指定座標を繰り返しクリック
- **複数点モード**: 複数座標を順番にクリック（周回）
- **クリック種別**: 左 / 右 / 中クリック
- **間隔設定**: 1〜600,000 ms
- **回数設定**: 無限 / 指定回数
- **座標取得**: 即時取得 / 3秒後に取得
- **グローバルホットキー**: 開始/停止 (F6)、停止 (F7)（変更可能）
- **設定保存/復元**: JSON形式で自動保存
- **タスクトレイ常駐**: ウィンドウを閉じてもバックグラウンド動作

## 必要環境

- Windows 10 / 11
- .NET 8.0 SDK

## ビルド

```bash
cd mouse_clicker
dotnet build AutoClicker.sln
```

## 単体 exe ファイルの作成（配布用）

.NET ランタイムを同梱した単一 exe を生成するには:

```bash
cd mouse_clicker
dotnet publish src/AutoClicker/AutoClicker.csproj -c Release -r win-x64 --self-contained -p:PublishSingleFile=true -p:IncludeNativeLibrariesForSelfExtract=true
```

出力先:

```
src/AutoClicker/bin/Release/net8.0-windows/win-x64/publish/AutoClicker.exe
```

この exe は .NET SDK/ランタイム未インストールの PC でもそのまま実行できます。

> **オプション**: ファイルサイズを小さくしたい場合は `-p:PublishTrimmed=true` を追加してください（約60〜80MB → 約20〜30MB）。
> ただしリフレクションに依存する処理がある場合はトリミングで動作が変わる可能性があるため、トリム後は必ず動作確認してください。

### ランタイム依存（軽量）版

配布先に .NET 8 ランタイムがインストール済みの場合は `--self-contained false` で軽量な exe を生成できます:

```bash
dotnet publish src/AutoClicker/AutoClicker.csproj -c Release -r win-x64 --self-contained false -p:PublishSingleFile=true
```

この場合の exe サイズは数MB程度です。

## 実行

```bash
dotnet run --project src/AutoClicker/AutoClicker.csproj
```

または、ビルド後に直接実行:

```bash
src/AutoClicker/bin/Debug/net8.0-windows/AutoClicker.exe
```

## 注意事項

- 間隔が極端に短い場合（例: 1ms）はOS負荷が高くなります
- マルチモニタ環境では座標が負になる場合があります
- クリック送出は専用ワーカースレッドで実行され、デッドロックを防止しています

## アーキテクチャ

`ARCHITECTURE.md` に従い、以下の安全設計を採用:

1. **専用ワーカー集約**: `SendInput` は `ClickEngine` 内の単一Taskでのみ実行
2. **多重起動禁止**: `SemaphoreSlim(1,1)` でガード
3. **busy-wait禁止**: `Task.Delay` + `CancellationToken` による即時停止対応
4. **ステートマシン**: Stopped → Running → Stopping → Stopped

## プロジェクト構成

```
src/AutoClicker/
  Native/          P/Invoke ラッパー (SendInput, GetCursorPos)
  Models/          データモデル (ConfigSnapshot, ClickPoint, AppSettings)
  Core/            ビジネスロジック (ClickEngine, HotkeyManager, SettingsStore)
  ViewModels/      MVVM ViewModel (MainViewModel, RelayCommand)
  Converters/      WPF値コンバーター
  MainWindow.xaml  メインUI
  App.xaml         アプリケーション定義
```

## 手動テスト手順

### 基本動作
1. アプリを起動し、ウィンドウタイトルが「Auto Clicker (Safe)」であることを確認
2. 初期状態: 状態=停止中、モード=単一点、間隔=100ms、回数=無限

### 単一点クリック
1. 「現在位置を取得」でX/Yに座標がセットされることを確認
2. 「3秒後に取得」で3秒待ってから座標取得されることを確認
3. 「開始」ボタンでクリックが始まり、状態が「実行中」になることを確認
4. 「停止」ボタンまたはF7で即座に停止することを確認

### 複数点クリック
1. 「複数点」ラジオを選択し、複数点設定が表示されることを確認
2. 「追加(現在位置)」でリストに座標が追加されることを確認
3. 「削除」「上へ」「下へ」が正しく動作することを確認
4. 開始・停止が正しく動作することを確認

### ホットキー
1. F6で開始/停止がトグルされることを確認
2. F7で停止のみ行われることを確認
3. 「変更」ボタンでホットキーを変更できることを確認
4. 同一キーの重複割り当てがエラーになることを確認
5. ESCでキャンセルできることを確認

### 設定保存
1. 設定を変更して終了
2. 再起動して設定が復元されていることを確認

### トレイ常駐
1. 「タスクトレイに常駐する」ON時、X閉じるでトレイに最小化されることを確認
2. トレイアイコンのメニュー（表示/開始停止/停止/終了）が動作することを確認
3. 「終了」ボタンは常駐設定に関係なくプロセスを終了することを確認

### エラー表示
1. 間隔に「abc」等の文字列を入力して「開始」→ 赤文字で「入力値を確認してください。」が表示されることを確認
2. 指定回数に0や-1を入力して「開始」→ エラー表示されることを確認
3. 正常値に戻して「開始」→ エラーが消え、実行開始されることを確認
4. 複数点モードでリスト空のまま「開始」→ エラー表示されることを確認

### 安全性（多重起動・連打耐性）
1. 実行中に「開始」を連打しても多重起動しないことを確認
   - ステータスバーが「実行中」のまま変化しないこと
2. F6を高速連打してもフリーズしないことを確認
   - 開始→停止が交互に安定して切り替わること
3. 不正な入力値（文字列、範囲外）で開始できないことを確認
4. 実行中にF7を連打しても正常に停止すること（二重停止でクラッシュしない）

### 停止の即応性
1. 間隔を5000ms（5秒）に設定して開始
2. 実行中に「停止」またはF7を押す → 5秒待たずに即座に停止すること
3. F6で停止しても同様に即座に止まること

### 実行中の設定変更
1. 実行中にモード切替ラジオ、間隔、回数の変更は自由に行える（UIはロックしない）
2. ただし実行中の値変更は現在の実行には影響しないこと（次回開始時に反映）

### ホットキー変更の回帰テスト
1. 「変更」ボタンを押してキャプチャモード → F6やF7を押しても実行が始まらないこと
2. キャプチャ中に任意のキーを押す → ホットキーが変更されること
3. 開始/停止キーに同じキーを割り当てようとする → エラー「ホットキーが重複しています。」が表示
4. LWin/RWin を割り当てようとする → エラー「ホットキーとして使用できないキーです。」が表示
5. ESCでキャプチャキャンセル → 変更されずに元のキーが維持されること
6. ホットキー変更後にF6/F7等の新キーでトグル/停止が正しく動作すること

### 設定保存・復元の回帰テスト
1. 間隔を1に変更、回数を指定回数で999999に変更して終了 → 再起動で復元を確認
2. settings.json を手動で `interval_ms: -100` に書き換えて起動 → 1にクランプされること
3. settings.json を手動で `interval_ms: 9999999` に書き換えて起動 → 600000にクランプされること
4. settings.json を手動で `hotkey_toggle: "LWin"` に書き換えて起動 → F6/F7にフォールバックされること
5. settings.json を削除して起動 → デフォルト値で正常起動すること

### 複数点モードの # 列番号
1. 複数点モードで3点以上追加 → # 列に 1, 2, 3… と1始まりで表示されること
2. 途中の行を削除しても番号が連番であること
3. 「上へ」「下へ」で行を移動しても番号が正しく振り直されること
