# CONTRIBUTING

このリポジトリは **Claude Code / Codex** など生成AIの併用を前提とします。
しかし、AIは親切心でUIを変更したり、入力送出の安全要件を崩しがちです。
本書はそれを防ぐためのルールです。

---

## 1. 重要: UI仕様は凍結
- UI/UXの唯一の正は `UI_SPEC.md` と `ui_contract.json`。
- **AI・人間を問わず**、以下は勝手に変更してはいけません:
  - 文言、初期値、ホットキー、レイアウト、表示/非表示、画面遷移、項目名
- UIを変更したい場合は「提案」として:
  1) `UI_SPEC.md` と `ui_contract.json` を先に更新するPRを作る（コード変更しない）
  2) レビュー承認後にだけ実装する

---

## 2. 重要: Windows 10デッドロック対策は最優先
- `ARCHITECTURE.md` の原則は必須。
- クリック送出（SendInput等）は専用ワーカーに集約し、多重起動禁止、busy-wait禁止。

---

## 3. AI（Claude Code / Codex）に与える共通ルール
AIへ渡す最初のプロンプトに必ず含めること:

- 「UIは凍結。`UI_SPEC.md` と `ui_contract.json` を絶対に変更しない」
- 「UI変更が必要ならコードを触らず、提案だけを書く」
- 「入力送出はワーカー集約。UI/ホットキーからSendInputしない」

推奨の短文（コピペ可）:
> 重要: UI仕様は凍結。UI_SPEC.md と ui_contract.json が唯一の仕様。UI/文言/初期値/ホットキー/レイアウトを一切変更するな。必要なら提案だけ書け。入力送出は専用ワーカー集約・多重起動禁止・busy-wait禁止。ARCHITECTURE.mdに従え。

---

## 4. ブランチ運用（推奨）
- `main`: 常に動く
- AI作業は `ai/<topic>` ブランチで行い、PRで統合
- PRは以下のチェック項目を満たすこと

---

## 5. PRチェックリスト（必須）
- [ ] UI_SPEC.md / ui_contract.json を変更していない（変更した場合はUI提案PRとして分離）
- [ ] ARCHITECTURE.md の原則（ワーカー集約・多重起動禁止・busy-wait禁止）に違反していない
- [ ] 開始/停止ホットキーを連打してもフリーズしない
- [ ] 連打中に開始を押しても多重起動しない（またはトグルで安全に停止する）
- [ ] 入力値バリデーションが働き、異常値で開始できない
- [ ] settings.json の保存/復元が壊れていない（デフォルト値に戻る等がない）

---

## 6. 変更禁止のファイル（原則）
- `UI_SPEC.md`
- `ui_contract.json`
- （追加するなら）`assets/screenshot.png` などUI比較用資産

---

## 7. 仕様の追加・変更手順
1) まず `UI_SPEC.md` / `ui_contract.json` / `ARCHITECTURE.md` のどれが影響するか整理
2) 仕様PR（ドキュメントのみ）を作る
3) 承認後に実装PRを作る（仕様PRと混ぜない）

---

## 8. 禁止事項まとめ
- UI凍結の破り（文言/初期値/ショートカット/レイアウトの勝手変更）
- UIやホットキーハンドラからの直接SendInput
- 実行ループの多重起動
- busy-wait
